# 小白版使用说明

## 这个软件是干嘛的

GalAssetBox 可以把你本地游戏文件夹里已经散放出来的图片、音乐、视频、文本脚本，自动按类别复制到一个新的整理文件夹。

它适合整理自己合法拥有或有授权处理的游戏/项目素材。

## 怎么用

浏览器版适合整理已经散放出来的图片、音乐、视频、文本。通用解包需要桌面版，因为它要调用你电脑上的 7-Zip、bsdtar 或 unar。

最常用流程：

1. 打开 `index.html`，或者启动桌面版。
2. 点 `选择游戏文件夹`，选择游戏本体所在的文件夹。
3. 点 `选择输出文件夹`，选择你想存放整理结果的位置。
4. 点 `扫描素材`。
5. 先看总览里的 `当前建议` 和 `整理前预检`。
6. 如果建议是 `开始整理`，确认数量没问题后直接点。
7. 如果建议是 `运行通用解包`，先运行；完成后点 `扫描解包输出`，再整理解出来的开放素材。

整理完成后，到输出文件夹里找 `GalAssetBox_整理结果_日期时间`。

桌面版整理完成后，总览页会出现 `打开结果文件夹` 按钮，可以直接打开刚刚生成的结果目录。

## 新手模式和高级功能

默认是 `新手模式`，只显示选择文件夹、扫描素材、开始整理这些常用按钮。

如果要导入插件、运行授权插件、生成清单或载入样例，点左侧的 `高级功能`。

## 怎么发群里求助

如果扫描结果看不懂，扫描后在 `总览` 页点 `给群友的求助摘要`。左侧 `高级功能` 里也有同样的导出入口。

它会生成一个 `.md` 文本文件，可以发给群友看。里面只有统计、封包类型、分类规则、插件状态和错误摘要，不包含图片、音乐、视频、文本正文，也不包含电脑里的绝对路径。

## 怎么微调分类

如果某个游戏的文件夹命名比较特殊，打开 `分类` 页，找到 `分类规则微调`。

例如：

- 输入 `tachie`，选择 `立绘`
- 输入 `event`，选择 `CG`
- 输入 `voice`，选择 `语音`

添加规则后，当前扫描结果会自动重新统计。`.xp3`、`.rpa` 这类封包仍然只会作为封包提示，不会因为规则变成可复制素材。

## 如果按钮灰色不能点

先看按钮下面的提示。常见原因有：

- 还没有选择游戏文件夹或输出文件夹
- 还没有点 `扫描素材`
- 当前是浏览器版，通用解包需要桌面版
- 没有发现普通压缩包候选
- 本机没有 7-Zip、bsdtar 或 unar
- 当前这一批普通压缩包已经尝试过，重新扫描会重置批次

总览里的 `当前建议` 会告诉你下一步优先做什么。

如果普通的 `选择文件夹` 按钮不能用，请换 Chrome 或 Edge 浏览器。也可以用下面这个方式打开：

```bash
cd GalAssetBox
python3 -m http.server 4174
```

然后在浏览器打开：

```text
http://localhost:4174
```

## 桌面版怎么试

如果你已经安装了 Node.js，可以在 `GalAssetBox` 文件夹里运行：

```bash
npm install
npm start
```

桌面版会弹出一个应用窗口，选择文件夹会更像普通软件。

如果你想生成一个本机可双击的应用目录，可以运行：

```bash
npm run pack
```

如果要打包 Windows 版本，可以看：

```text
docs/WINDOWS.md
```

在这台 macOS Apple Silicon 机器上，生成的位置是：

```text
dist/mac-arm64/GalAssetBox.app
```

如果你不懂这些命令，先继续用 `index.html`；等正式发版本时，可以把桌面版打成更普通的安装包。

## 为什么有些素材没有被提取出来

很多 galgame 会把素材放进两类包里：

- 普通外层压缩包，例如 `.zip`、`.rar`、`.7z`、`.tar`
- 引擎资源封包，例如 `.xp3`、`.rpa`、`.nsa`、`.pck`、`.dat`、`.pak`、`.assets`、`.resS`

桌面版可以把普通外层压缩包交给本机工具做“通用解包”。如果 `.dat`、`.pak` 这类文件其实是 ZIP/RAR/7z 签名，也会按普通压缩包路线提示。

通用解包一次最多处理 12 个候选。处理完成后可以直接点 `扫描解包输出`，把解出来的文件继续按 CG、立绘、音乐、视频、文本分类整理。

通用解包报告会写清楚候选总数、本轮处理数量、未处理预览、普通失败和策略拦截。策略拦截通常表示工具解出了符号链接、越界路径或超过安全限额，软件会停下来让你人工确认。

引擎资源封包会进入封包页的路线提示。GalAssetBox 不会破解、解密、绕过 DRM，也不会内置第三方游戏密钥。它能直接整理的是文件夹里已经明文看得到的图片、音乐、视频、文本等开放文件。

## 授权插件是什么

有些项目是自制游戏、开源游戏，或者作者/团队明确允许你处理资源。以后可以用“授权插件”来适配这种格式。

授权插件不会默认破解游戏，也不会内置别人的密钥。它只适合你自己有权处理的项目。

## 怎么运行授权插件

1. 先选择游戏文件夹。
2. 再选择输出文件夹。
3. 点 `扫描素材`。
4. 打开 `插件` 页，确认是否有“可执行”的匹配项。
5. 点 `运行授权插件`。
6. 看清弹窗说明后再确认。

插件结果会放在新的 `GalAssetBox_授权插件结果_日期时间` 文件夹里。

## 怎么开关插件

打开 `插件` 页，每个插件卡片上都有 `启用插件` 或 `停用插件`。

这个开关只保存在你当前浏览器里，不会上传，也不会改游戏文件。

## 怎么导入外部插件

1. 打开 `插件` 页。
2. 点 `导入本地插件`。
3. 选择可信的 `.js` 或 `.gabplugin.js` 插件文件。
4. 看清弹窗再确认。

外部插件相当于本地代码，只导入你信任的人给你的插件。不要导入来源不明的插件。

## 怎么导入插件包

1. 打开 `插件` 页。
2. 点 `导入插件包`。
3. 选择包含 `plugin.json` 的插件文件夹。
4. 看清弹窗再确认。

插件包只是更整齐的插件文件夹，里面通常有说明、入口脚本和测试样例。它同样只适合可信来源和授权项目。

## 插件安装区怎么看

`插件安装区` 会显示每个插件从哪里来：

- `内置`：软件自带
- `单文件`：你刚刚导入的 `.js` 插件
- `插件包`：你刚刚导入的插件文件夹

它还会显示版本、作者、入口文件和导入时间，方便你确认有没有导错。

## 怎么跑一遍示例

这个示例只是测试软件流程，不含任何游戏素材。

1. 点 `选择游戏文件夹`，选择 `examples/plugin-package-demo`。
2. 点 `选择输出文件夹`，选择一个临时文件夹。
3. 打开 `插件` 页。
4. 点 `导入插件包`，还是选择 `examples/plugin-package-demo`。
5. 点 `扫描素材`。
6. 回到 `插件` 页，确认“真实试跑向导”里的步骤已经变成完成。
7. 点 `运行授权插件`。
8. 到输出文件夹里找 `GalAssetBox_授权插件结果_日期时间`。

如果成功，你会看到插件生成的 `package-demo/summary.txt`。

跑完以后，插件页会出现 `最近输出结果`，里面会告诉你：

- 结果文件夹叫什么
- 报告文件在哪里
- 清单文件在哪里
- 插件输出文件在哪里

## 怎么导出诊断包

如果软件出了问题，扫描后在 `总览` 页点 `给开发者的诊断包`。插件页也保留了诊断包入口。

它会下载一个 `.json` 文件，里面只有排查用的信息，比如插件状态、扫描清单、日志和错误信息。

诊断包不会包含图片、音乐、视频、文本正文，也不会包含电脑里的绝对路径。

## 整理后的文件在哪里

在你选择的输出文件夹里，例如：

```text
GalAssetBox_整理结果_20260513_0030/
```

里面会有：

- `01_CG`
- `02_立绘`
- `03_背景`
- `05_音乐`
- `08_视频`
- `09_文本脚本`
- `GalAssetBox_整理报告.md`
- `GalAssetBox_素材清单.csv`

## 注意

不要把整理出来的版权素材到处传播。

这个工具只适合个人备份、素材管理、合法授权项目、翻译调试等场景。
